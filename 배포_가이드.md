# 웹사이트 배포 가이드 (프로그래매틱 SEO)

> 목적: ① 개인정보처리방침 URL 확보 (Play Console 필수) ② 검색 노출용 키워드 랜딩 페이지

## 구성
- `index.html` — 메인 랜딩 (구조화 데이터 JSON-LD 포함 → 구글 검색에 앱 정보 리치 표시)
- 키워드 페이지 6개 — 블록 쌓기 / 탑 쌓기 / 스택 / 아이소메트릭 / 캐주얼 / 타이밍 게임 (각각 고유 콘텐츠)
- `privacy.html` — 개인정보처리방침 (Play Console 등록용)
- `sitemap.xml` + `robots.txt`

## 배포 (GitHub Pages, 무료 — 5분)
1. github.com 로그인 (taewoong3) → New repository → 이름 `stack-tower` → Public 생성
2. 이 `website/` 폴더 안의 파일 전부를 리포지토리에 업로드 (웹에서 "uploading an existing file" 드래그)
3. Settings → Pages → Branch: `main`, 폴더 `/` → Save
4. 몇 분 후 `https://taewoong3.github.io/stack-tower/` 접속 확인

## 배포 후 할 일
- [ ] Play Console 개인정보처리방침 URL에 입력: `https://taewoong3.github.io/stack-tower/privacy.html`
- [ ] [Google Search Console](https://search.google.com/search-console) 에 사이트 등록 → sitemap.xml 제출 (검색 노출 시작)
- [ ] 앱 출시 후: 스토어 링크가 이미 모든 페이지에 심어져 있음 (패키지명 기준) — 별도 수정 불필요
- [ ] 커뮤니티 출시 글에 스토어 링크와 함께 이 사이트 링크도 첨부 (백링크 = 검색 순위 상승)

## 유지보수
- 계정 아이디가 taewoong3이 아니면 각 html의 `taewoong3.github.io` 주소만 일괄 치환
- 키워드 페이지 추가는 기존 페이지 하나 복사 → 제목/본문/캐노니컬만 수정 → sitemap.xml에 한 줄 추가
